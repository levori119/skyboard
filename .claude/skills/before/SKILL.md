# Gate לפני קוד — SKY-KING

לפני כל שורת קוד, עבור על הרשימה הזו. אל תדלג.

## קרא CLAUDE.md

קרא את `CLAUDE.md`. זכור: אנחנו בונים מחליף לסדק פיזי של בקרי טיסה.

---

## Checklist — חובה לעבור לפני קוד

### 1. הבנת המשימה
- [ ] האם הפעלתי `/pm` וקיבלתי acceptance criteria ברורים?
- [ ] האם הפעלתי `/arch` וקיבלתי תכנית טכנית?
- [ ] האם ה-CEO אישר את הגישה?

### 2. DRY — לא לשכפל
- [ ] חיפשתי ב-`App.tsx` — האם יש רכיב דומה?
- [ ] חיפשתי ב-`server.js` — האם יש route דומה?
- [ ] אם מצאתי — האם אני מרחיב במקום יוצר חדש?

### 3. DB
- [ ] האם נדרש שינוי סכמה? → הפעל `/migrate` קודם
- [ ] האם הפעלתי `/migrate` אם נדרש?

### 4. השפעה על מסכים אחרים
- [ ] האם שינוי ברכיב משותף? → רשמתי אילו מסכים יושפעו
- [ ] בדקתי גם את CTRL view וגם את TWR (מגד/מגרש)?

### 5. סטנדרטים
- [ ] כל טקסט UI שאכתוב יהיה בעברית
- [ ] Event Log — ידעתי אם נדרש רישום
- [ ] אין secrets בקוד

---

## אם checklist עבר — אפשר להתחיל

ציין בקצרה: "Gate עבר ✅ — מתחיל לכתוב [שם הפיצ'ר]"

## אם checklist נכשל

ציין מה חסר ועצור. אל תתחיל לכתוב קוד.
