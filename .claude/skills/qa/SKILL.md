## שלב 2.5 — 🚦 RUNTIME VERIFICATION GATE (חובה — לא לדלג)

> **`tsc` + `build` + unit tests מוכיחים שהקוד מתקמפל ושהלוגיקה הטהורה נכונה — הם
> *בעקרון* לא יכולים לתפוס באגים של runtime / ויזואלי / הנחה ארכיטקטונית שגויה.**
> כל הבאגים האמיתיים שלנו עברו את הבדיקות הסטטיות: API 404 (קידומת כפולה),
> כלי "או"ק" שצייר מעגל (ניתוב handler), ציורים שלא זזים עם המפה (הנחה שגויה על
> ה-DOM). **בדיקות סטטיות = הכרחי, לא מספיק.**

**שתי שכבות חובה לפני "done":**

1. **Static gate:** `tsc --noEmit` + `npm test` + `vite build` — כולם ירוקים.
2. **Runtime gate לפי סוג השינוי — להריץ ולצפות בפועל, לא להסיק/להניח:**
   | סוג | אימות חובה |
   |-----|------------|
   | **UI / מפה / תצוגה** | להריץ את האפליקציה ולבצע את הפעולה בפועל (`/run` או `/verify`); למפה — לבדוק גם **pan / zoom / שינוי גודל מסך** |
   | **API / backend** | smoke test ל-endpoints מול נתונים אמיתיים (200 + תוכן נכון) |
   | **DB / SQL** | להריץ מול rows אמיתיים כולל edge cases |
   | **ביצועים** | בדיקת עומס / מדידת latency |

3. **לאמת הנחות ארכיטקטוניות מול הקוד/DOM בפועל** — אם הפיצ'ר נשען על הנחה
   ("הקנבס בתוך ה-transform", "ה-route כבר קיים") — **לאמת אותה בקוד לפני שמסמנים done**.
   הנחה לא-מאומתת = ממצא קריטי.

**אם אי-אפשר להריץ אינטראקטיבית (login/מפה/חומרה):**
→ סמן **❌ לא אומת ויזואלית — חובה בדיקת משתמש** כ**ממצא קריטי (BLOCKER)**, *לא* כ-⚠️ צדדי.
ה-done מותנה באישור המשתמש שראה את זה עובד.

## שלב 3 — Checklist מלא

עבור על כל הסעיפים הבאים ותן ✅/❌/⚠️ לכל אחד:

### UI & שפה
- [ ] כל טקסט UI בעברית (ללא אנגלית נסתרת בtooltips, placeholders, error messages)
- [ ] Dark mode — הרכיב נראה טוב ברקע כהה
- [ ] פונטים גדולים, ניגודיות — קריאה ממרחק (מסך בקרה)
- [ ] RTL תקין — אין אלמנטים שהתהפכו לא נכון
- [ ] אישור לפני מחיקה / פעולה קריטית

### פונקציונליות
- [ ] הפיצ'ר עושה בדיוק מה שה-acceptance criteria אמרו (בדוק מול `/pm`)
- [ ] Edge cases: מה קורה עם פ"מ ריק? עם שדות חסרים? עם DB לא זמין?
- [ ] שינוי ברכיב משותף — נבדק על כל המסכים שמשתמשים בו

### נתונים & DB
- [ ] Event Log — כל שינוי סטטוס רשום ב-activity_log
- [ ] אין מחיקת היסטוריה (soft delete או log בלבד)
- [ ] JSONB fields — מבנה תקין, לא string גולמי

### קוד
- [ ] אין רכיב חדש שכפל רכיב קיים (DRY)
- [ ] אין secrets/tokens בקוד
- [ ] אין console.log שנשאר
- [ ] אין TODO שנשאר

### ביצועים & real-time
- [ ] הפיצ'ר עובד עם polling הנוכחי (5 שניות)
- [ ] אין N+1 queries מיותרות

## כללים
- אל תתקן בעצמך — תדווח בלבד. CEO מחליט אם לתקן עכשיו או לדחות.
- אם מצאת בעיה שגורמת לנזק תפעולי (לדוגמה: מחיקת נתון ללא אישור) — זה BLOCKER.
- **אסור לסמן ✅ "מוכן לשחרור" בלי שעבר ה-Runtime Verification Gate.** "מתקמפל + בדיקות עוברות" לבדו = מקסימום ⚠️ שחרור עם הסתייגות, לעולם לא ✅.
