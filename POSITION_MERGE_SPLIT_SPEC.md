# איחוד ופיצול עמדה — אפיון מלא (Position Combine / Decombine)

> סטטוס: **אפיון לאישור — אין קוד מימוש עדיין.** ליבת הלוגיקה הטהורה כבר קיימת ב-`src/utils/unifiedStrips.ts` (+ בדיקות).
> נוגע במנגנון ההעברות והתצוגה → לעבוד תחת `/transfer-logic` + `/qa` קפדני בין פרוסות.

---

## 0. הגדרות (מינוח — לא פ"מ!)

- **איחוד עמדה (Combine):** בקר A לוקח אחריות מבצעית על עמדה B (וכו') — בעומס נמוך. A **רואה ומתפעל** את כל מה ששייך ל-B.
- **פיצול עמדה (Decombine):** A משחרר את B בחזרה לבקר שלה — בעומס גבוה. תצוגת A חוזרת למצבה.
- **הפ"מים = נגזרת בלבד:** האיחוד הוא של ה**עמדה**; אילו פ"מים זורמים פנימה/החוצה זו תוצאה. `unifiedStrips.ts` עונה רק על *"אילו פ"מים בתצוגה שלי כתוצאה מאיחוד B"*.

## 2. ארכיטקטורה

### 2.1 DB (מיגרציה דרך `/migrate` → עדכון `data-model.md`)
טבלה `position_merges`:
```
id              SERIAL PK
covering_preset_id  INT  NOT NULL   -- העמדה שמאחדת (A)
covered_preset_id   INT  NOT NULL   -- העמדה שמאוחדת (B)
started_by      INT               -- crew_member שביצע
started_at      TIMESTAMPTZ NOT NULL DEFAULT now()
ended_at        TIMESTAMPTZ        -- NULL = פעיל
UNIQUE(covered_preset_id) WHERE ended_at IS NULL   -- B מכוסה ע"י לכל היותר עמדה אחת
```
> **חובה TIMESTAMPTZ** (כלל הפרויקט — REFACTOR_LOG #016). פעיל = `ended_at IS NULL`.

### 2.2 שרת (`server/routes/position-merges.js`)
- `GET  /api/position-merges?active=1` — איחודים פעילים (לכל הלקוחות).
- `POST /api/position-merges` `{covering, covered}` — איחוד. ולידציות: §4.
- `PATCH /api/position-merges/:id/end` — פיצול (מציב `ended_at`).
- כל שינוי → **broadcast real-time** (אותו ערוץ כמו strips/transfers).

### 2.3 לקוח (`SectorDashboard.tsx`)
1. fetch + real-time של איחודים פעילים שבהם `covering = my preset` → רשימת עמדות מכוסות.
2. בניית `CombinedPosition[]` לכל עמדה מכוסה: `{presetId, filter: their filter_query, ctx:{presetId, presetName, aviationBases}}`.
3. הזנת ה-union לתוך חישוב `myTableStrips` (לא לשכפל — לעטוף).
4. אם `covered_preset_id == my preset` ופעיל → **נעילה**: "העמדה מכוסה ע"י {A}" — read-only/redirect (מונע תפעול כפול).

### 2.4 לוגיקה טהורה (`unifiedStrips.ts`) — הרחבות נדרשות
הקובץ הנוכחי מטפל ב-query-union בלבד. להרחיב ל**סט-עמדה מלא** (החלטה 4):
```
stripsForPosition(strips, position) =
   query match  ∪  workstation_preset_id == B  ∪  table_preset_ids ∋ B  ∪  pending_transfer נכנס ל-B
unifiedSet = dedupeByParent( stripsForPosition(me) ∪ ⋃ stripsForPosition(Bᵢ) )
```
תוספות: **dedup לפי `parent_strip_id`** (פיצול חנית בין עמדות), **cycle-guard**, **התעלמות מ-self/כפילות**.

### 2.5 מפה (החלטה 3 — DRY מלא)
- עמדה מאוחדת B עם **אותה מפה** כמו A → הפ"מים שלה על המפה הקיימת, מתויגים (מקור-עמדה).
- B עם **מפה אחרת** → מוצגת כ-**map2 דינמי** (`map2_id := B.map_id` בזמן ריצה), עם נקודות העברה/אזורים של B. **אפס קוד תצוגה חדש** — תשתית dual-map הקיימת.
- פיצול → הסרת ה-map2 הדינמי.

## 3. תפעול (החלטה 1) — סמנטיקת בעלות ופעולות
- פ"מ מאוחד נשאר בבעלות `workstation_preset_id = B` עד העברה.
- פעולה (העברה/עריכה/סטטוס) ע"י A על פ"מ של B → **actor = crew של A**, נרשם ב-`activity_log` כ"A בשם B".
- נעילת B (§2.3.4) מבטיחה שאין שני מתפעלים על אותו פ"מ.
- העברה החוצה מפ"מ מאוחד → flow רגיל (`pending_transfer`/accept), ללא שינוי.

## 4. מקרי קצה / ולידציות (חובה לכסות)
1. **מעגל:** A מאחד B, B כבר מאחד A → דחייה.
2. **כיסוי כפול:** B כבר מכוסה ע"י C → דחייה (UNIQUE).
3. **איחוד עצמי / כפילות:** `covered == covering` או כבר ברשימה → דחייה/no-op.
4. **דדופ חנית:** חנית שפוצלה בין A ל-B → שורה אחת (dedup parent).
5. **הצפה:** עמדה מאוחדת בלי filter → רק בעלות-preset (לא כל הפ"מים). ✅ כבר נבדק.
6. **real-time:** שינוי `filter_query` של B / העברה ב-B → תצוגת A מתעדכנת.
7. **התראות:** האם פ"מ מאוחד נספר ב-load/conflict alerts של A? → **כן** (תפעול = חלק מהתמונה).
8. **B מתנתק/נכנס תוך כדי כיסוי:** נעילה מונעת; בכניסה רואה "מכוסה".
9. **חפיפה ל-full-picture:** "תמונה מלאה" = צפייה בכל הקבוצה (read-only). "איחוד" = תפעול סלקטיבי. **שני מצבים נפרדים — לא לשכפל.**

## 5. פיצול עמדה (Decombine) — אפיון

### 5.1 הגדרה
A (מכסה) משחרר עמדה מכוסה B. אין תזוזת נתונים — רק סגירת רשומת האיחוד.

### 5.2 זרימה
1. A לוחץ "פצל עמדה B" (או "פצל הכל").
2. `PATCH /position-merges/:id/end` → `ended_at = now()`.
3. broadcast → תצוגת A מחשבת `combined[]` מחדש (B יורד), map2 דינמי של B מוסר.
4. נעילת B מוסרת → בקר B יכול לחזור לתפעל.

### 5.3 כללים
- **פיצול חלקי:** A מכסה B+C → יכול לפצל רק C, נשאר עם B.
- **מי מפצל:** A (המכסה), או admin.
- **פ"מים שנערכו תוך כיסוי:** נשארים בבעלותם הנוכחית (B או יעד ההעברה). שום נתון לא "חוזר".
- **פיצול אוטומטי:** ביציאת A / סוף משמרת → לסגור איחודים פעילים שלו (לא להשאיר B נעולה לתמיד). **נגזר מ-§4.8.**
- **אידמפוטנטי:** פיצול עמדה שכבר מפוצלת = no-op.

### 5.4 מקרי קצה לפיצול
- פיצול תוך כדי שלפ"מ של B יש `pending_transfer` פתוח שיזם A → ההעברה נשארת תקפה (היא על הפ"מ, לא על האיחוד).
- A מתנתק בלי לפצל → §5.3 פיצול אוטומטי.
- B כבר לא קיים (נמחק preset) → לסגור איחוד בשקט.

## 6. תהליכי בדיקה (4 שכבות)

### א. Unit (`unifiedStrips.test.ts` — קיים 11, להרחיב)
- ✅ קיים: own-filter, union, fallback בעלות, dedup-inclusion, סדר.
- ➕ להוסיף: `stripsForPosition` (query+בעלות+table_preset_ids+pending), **dedup-parent**, **cycle-guard**, self/כפילות, פיצול (combined יורד → זהה ל-myStrips).

### ב. Integration (חדש)
harness עם מערך `strips` ריאלי + merges → אימות `myTableStrips` המורחב. **בדיקת אי-נסיגה:** `combined=[]` ⇒ פלט זהה בית-לבית למצב היום.

### ג. Manual QA — מטריצה
| תרחיש | ציפייה |
|---|---|
| איחוד B (עם filter) | פ"מי B מצטרפים, בלי כפילות, מתפעלים |
| איחוד B (בלי filter) | רק בעלות B |
| B עם מפה אחרת | map2 דינמי עם נק' העברה של B |
| פיצול חנית A↔B | שורה אחת |
| העברה מפ"מ מאוחד | בעלות + log = "A בשם B" |
| נעילת B | בקר B רואה "מכוסה ע"י A" |
| פיצול B | חזרה מדויקת; B משוחרר |
| פיצול חלקי (B+C→split C) | נשאר B |
| ניתוק A בלי פיצול | פיצול אוטומטי |
| real-time: שינוי ב-B | תצוגת A מתעדכנת |

### ד. Regression + Performance
- אי-נסיגה: `myStrips`/`fullPicture`/transfers ללא שינוי כשאין איחוד.
- ביצועים: union O(N·M) (פ"מים × עמדות מאוחדות) — לוודא סביר ל-N,M ריאליים.
- TIMESTAMPTZ במיגרציה.

## 8. סיכונים פתוחים
- תפעול כפול אם הנעילה נכשלת (real-time latency) → לשקול נעילה אופטימית בשרת.
